### Short Description (for cards)

### Full Description (for detail page)

- Real-time financial data validation and reconciliation
- Comprehensive QA and defect tracking system

### Impact
- Eliminated manual data entry for finance operations
- Zero-discrepancy ERP integration across all transaction types
- Reduced reconciliation preparation time for finance team
- Handles 50+ line items per order with automated validation

## Project 3: API Bank Reconcile ⭐

### Basic Info
- **Title**: API Bank Reconcile
- **Subtitle**: Bank Statement Parser & Reconciliation API
- **Period**: May 2026
- **Status**: Deployed (Railway)

### Tech Stack
Next.js • React • TypeScript • PostgreSQL • Railway

### Short Description
REST API service that parses bank statements (PDF, HTML, CSV) and returns normalized JSON for reconciliation systems, supporting BCA, BRI, and Mandiri formats.

### Full Description
Designed and deployed a REST API service that ingests raw bank statement files in multiple formats (PDF, HTML, CSV) and returns standardized JSON output for downstream reconciliation systems.

The API handles bank-specific parsing logic for three major Indonesian banks (BCA, BRI, Mandiri), normalizing disparate data formats into a unified schema that eliminates parsing errors and ensures consistent data structure across all supported banks.

### Key Features
- Multi-format support (PDF, HTML, CSV)
- Bank-specific parsing logic (BCA, BRI, Mandiri)
- Unified JSON output schema
- RESTful API design
- Deployed on Railway with auto-scaling
- Error handling and validation

### Impact
- Standardized bank statement data across 3+ banks
- Eliminated manual parsing errors
- Reduced reconciliation preparation time
- Scalable API infrastructure for future bank additions
